# Project Structure

This document outlines the directory organization for the ElevenLabs Unity Plugin (`FF.ElevenLabs`).

## Directory Overview

*   **`Assets/ElevenLabs/`**: Root folder for the plugin.

    *   **`Runtime/`** (`ElevenLabs.Runtime.asmdef`)
        *   Contains core logic that needs to run in the built game.
        *   **Key Files**:
            *   `VoiceModule.cs`: ScriptableObject definition for voice groupings.
            *   `VoiceStep.cs`: Data class for individual voice lines.
            *   `ElevenLabsModels.cs`: JSON serializable classes for API communication.
        *   **Namespace**: `FF.ElevenLabs`

    *   **`Editor/`** (`ElevenLabs.Editor.asmdef`)
        *   Contains all Editor-only logic, Windows, and custom inspectors.
        *   **Key Files**:
            *   `ElevenLabsEditorWindow.cs`: The main UI for the tool (`Window > Voice Over`).
            *   `ElevenLabsAPI.cs`: Handles HTTP requests to the ElevenLabs API.
            *   `ElevenLabsUtilities.cs`: Helper functions for key management and obfuscation.
        *   **Namespace**: `FF.ElevenLabs.Editor`

    *   **`Resources/`**
        *   Contains the `ElevenLabsConfig.asset` singleton used to store project-wide settings (like default voice stability).

    *   **`VoiceModules/`**
        *   Default location for user-created `VoiceModule` assets.

    *   **`Generated/`** (Git Ignored)
        *   Stores the actual `.wav` audio files generated by the plugin.
        *   Organized by Module Name.
        *   *Note: This folder is excluded from version control to save space.*

## Key Design Principles

1.  **Assembly Separation**: Code is strictly divided into `Runtime` and `Editor` assemblies to prevent build errors and ensure cleaner dependency management.
2.  **Asset-Based Workflow**: The plugin uses `ScriptableObjects` (`VoiceModule`) as the source of truth for voice lines, allowing for easy serialization and version control of text content.
3.  **Secure Storage**: API keys are stored in `EditorPrefs` (local machine only), not in asset files, to prevent accidental leaks in version control.
